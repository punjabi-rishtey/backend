# Punjabi Rishtey - Backend

This repository contains the backend for the Punjabi Rishtey matrimonial platform. It serves the necessary APIs for user authentication, managing profiles, handling memberships, testimonials, messaging, and admin operations.

It is designed to be consumed by the **user frontend**, the **admin frontend**, and any related client applications.

## 🚀 Tech Stack

- **Runtime:** Node.js
- **Framework:** Express.js
- **Database:** MongoDB & Mongoose
- **Authentication:** JWT (JSON Web Tokens) & bcryptjs
- **Media Storage:** Cloudinary (via multer-storage-cloudinary)
- **Email:** Nodemailer
- **API Documentation:** Swagger (`swagger-ui-express`)
- **Task Scheduling:** node-cron

---

## 💻 Local Setup & Development

### Prerequisites
- Node.js (v18+ recommended)
- MongoDB (local or Atlas URI)

### Installation

1. **Clone the repository** and navigate to the backend directory:
   ```bash
   cd punjabi-rishtey/backend
   npm install
   ```

2. **Setup Environment Variables**
   Create a `.env` file in the root directory and populate it with the required keys:
   ```env
   # Typical .env values required
   PORT=8080
   MONGO_URI=your_mongodb_connection_string
   JWT_SECRET=your_jwt_secret

   # Cloudinary for Image Uploads
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=your_api_key
   CLOUDINARY_API_SECRET=your_api_secret

   # Email Service
   EMAIL_USER=your_smtp_email
   EMAIL_PASS=your_smtp_password

   # Frontend Allowed Origins
   FRONTEND_URL=http://localhost:5173
   ```

3. **Run the Development Server**
   ```bash
   npm run dev
   ```
   The backend will typically start on `http://localhost:8080`.

## 🔗 Main Route / Domain Map

The API is logically divided into domains under `/api`:

- `/api/users`: User signup, login, profile updates, and fetching profiles.
- `/api/families`: Family details specific to matrimonial profiles.
- `/api/educations`: Education metrics associated with user profiles.
- `/api/professions`: Professional details associated with user profiles.
- `/api/astrologies`: Astrological matching information.
- `/api/memberships`: Logic for listing and assigning membership plans.
- `/api/coupons`: Coupon code validation and processing.
- `/api/testimonials` & `/api/review`: Managing platform reviews and success stories.
- `/api/messages`: Intra-platform messaging logic.
- `/api/admin/auth`: Authentication and specialized routes for the admin dashboard.

## 🛠 Operational & Architecture Notes

### Auth Flows
Authorization is stateless. We use `jsonwebtoken` (JWT). Upon `/api/users/login`, the backend issues a token that the frontend sends in an `Authorization: Bearer <token>` header for protected routes.

### Media Uploads
Files (like user profile pictures) are managed through `multer`. The current configuration pipes uploads directly to Cloudinary using `multer-storage-cloudinary`, preventing images from filling up the local/ephemeral disk spaces.

### Cron Jobs
Membership expiries and status adjustments are processed by a cron job located at `utils/checkExpiryCron.js`. **Note:** This job does not currently run inside the main `index.js` web process. It needs to be executed separately via `npm run start:cron` (i.e., on a Render Background Worker).

### API Documentation (Swagger)
Interactive API documentation is generated by Swagger. While the server is running, navigate to:
`http://localhost:8080/api-docs` to view and test API endpoints.

### CORS & Allowed Origins
CORS is explicitly configured in `index.js` to whitelist specific domains (e.g., Vercel frontend environments, Admin URLs, and `localhost:5173`/`5174`). Ensure your frontend origin is added to this list or your `FRONTEND_URL` in `.env` if requests are blocked.

## ☁️ Deployment Notes

- **Platform:** Currently configured for deployment on **Render**.
- **Services:** Usually deployed as a Web Service running `npm start`.
- **Background Worker:** The `start:cron` command will ideally run as a Background Worker service on Render to consistently process daily expirations without blocking the web process.
